Command Injection Techniques

Command Separators

Command injection often relies on special characters to separate commands:

  • Semicolon (;): Executes commands sequentially regardless of previous command success
  • AND (&&): Executes the second command only if the first succeeds
  • OR (||): Executes the second command only if the first fails
  • Pipe (|): Pipes the output of the first command as input to the second
  • Background (&): Runs the command in the background

Command Injection Types

  • Direct Injection: Command output is directly returned in the HTTP response
  • Blind Injection: No direct output; detection through time delays or out-of-band techniques
  • Out-of-Band Injection: Uses external channels (DNS, HTTP) to exfiltrate data

Common Attack Payloads

  • Information Gathering: ; whoami, && id, | uname -a
  • File System Access: ; ls, && dir, | cat /etc/passwd
  • Network Reconnaissance: ; ifconfig, && ipconfig, | netstat -an
  • Time-based Detection: ; sleep 10, && timeout 10

Remediation Recommendations

To prevent Command Injection vulnerabilities, implement the following measures:

Input Validation and Sanitization

  1. Whitelist Approach: Only allow known good input patterns
  2. Blacklist Approach: Block dangerous characters and command separators
  3. Input Length Limits: Restrict input length to prevent complex payloads
  4. Character Encoding: Properly encode special characters

Secure Coding Practices

  1. Avoid Shell Execution: Use secure APIs instead of system calls when possible
  2. Parameterized Commands: Use parameterized command execution functions
  3. Privilege Separation: Run applications with minimal privileges
  4. Output Encoding: Encode command output before displaying

Web Server Configuration

  1. Disable Dangerous Functions: Disable shell execution functions in PHP/other languages
  2. Web Application Firewall: Implement WAF rules to detect command injection attempts
  3. Request Monitoring: Monitor and log suspicious command-like input
  4. Rate Limiting: Implement rate limiting to prevent automated attacks
