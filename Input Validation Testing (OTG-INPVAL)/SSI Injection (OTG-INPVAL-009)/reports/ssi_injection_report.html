SSI Injection Assessment

DVWA Security Test - OWASP Testing Guide OTG-INPVAL-009

Executive Summary

This assessment identified 21 SSI Injection vulnerabilities in the DVWA application. Server-Side Includes (SSI) Injection occurs when applications process user input as SSI directives, potentially allowing attackers to execute commands, include files, or access sensitive server information.

Methodology

The testing methodology followed the OWASP Testing Guide for SSI Injection:

  1. Authentication: Logged into DVWA with provided credentials
  2. Endpoint Discovery: Identified input endpoints throughout DVWA modules
  3. SSI Testing: Tested various SSI injection payloads including command execution, file inclusion, and variable echoing
  4. Analysis: Documented endpoints that processed SSI directives in unexpected ways

SSI Injection Techniques

The following SSI injection techniques were tested:

  • Command Execution: Using to execute system commands
  • Variable Echoing: Using to display server variables
  • Date/Time Injection: Using to manipulate date output
  • File Inclusion: Using to include file contents

SSI injection can lead to remote code execution, information disclosure, and server compromise.

Remediation Recommendations

To prevent SSI Injection vulnerabilities, implement the following measures:

  1. Disable SSI: Disable Server-Side Includes if not required by the application
  2. Input Validation: Validate and sanitize all user input, especially for characters like
