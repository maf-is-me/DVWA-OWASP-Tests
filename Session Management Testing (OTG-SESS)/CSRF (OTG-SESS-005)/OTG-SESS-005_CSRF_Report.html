Executive Summary

This report details the findings of automated testing for Cross-Site Request Forgery (CSRF) vulnerabilities (OTG-SESS-005) on the DVWA application. CSRF is an attack that forces an end user to execute unwanted actions on a web application in which they're currently authenticated. CSRF attacks specifically target state-changing requests, not theft of data.

Methodology

The following methodology was used to test for CSRF vulnerabilities:

  • Logged into DVWA with valid credentials
  • Tested CSRF protection at different security levels (Low, Medium, High)
  • Analyzed forms for the presence of anti-CSRF tokens
  • Attempted to submit forms without valid CSRF tokens
  • Tested state-changing operations (security level change, password change)
  • Verified server-side validation of CSRF tokens

Remediation Recommendations

  • Implement unique, unpredictable CSRF tokens for all state-changing forms
  • Validate CSRF tokens on the server side for every request
  • Regenerate CSRF tokens after successful form submission
  • Use the SameSite cookie attribute to prevent CSRF attacks
  • Consider implementing double submit cookies pattern as additional protection
  • Set appropriate security headers (X-Frame-Options, Content-Security-Policy)
  • Perform regular security code reviews to identify CSRF vulnerabilities
