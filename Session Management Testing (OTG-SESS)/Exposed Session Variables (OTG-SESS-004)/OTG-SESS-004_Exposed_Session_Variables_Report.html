Executive Summary

This report details the findings of automated testing for Exposed Session Variables (OTG-SESS-004) on the DVWA application. The tests focused on identifying session-related information that is unnecessarily exposed to the client-side, which could potentially be exploited by attackers to hijack sessions or gain unauthorized access.

Risk Rating: HIGH

Findings: 164

Test Details

Methodology

The following methodology was used to test for exposed session variables:

  • Logged into DVWA with valid credentials
  • Navigated through multiple application pages
  • Analyzed HTML source code for session-related variables
  • Checked URLs for session data in query parameters
  • Scanned JavaScript code for exposed session tokens
  • Reviewed HTML comments and meta tags for sensitive data
  • Identified hidden form fields containing session information

Remediation Recommendations

  • Avoid passing session identifiers in URLs or query parameters
  • Do not store session tokens in client-side JavaScript variables
  • Remove session-related information from HTML comments and meta tags
  • Ensure hidden form fields do not contain sensitive session data
  • Implement proper server-side session management
  • Use HTTPOnly and Secure flags for session cookies
  • Regularly audit application code for exposed sensitive information
  • Implement Content Security Policy (CSP) to restrict script execution
