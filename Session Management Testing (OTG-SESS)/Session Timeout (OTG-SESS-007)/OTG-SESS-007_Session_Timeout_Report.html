Remediation Recommendations

  • Implement appropriate session timeout periods (5-30 minutes for web applications)
  • Properly invalidate sessions on the server side when they expire
  • Redirect users to the login page after session timeout
  • Destroy all session data associated with expired sessions
  • Prevent reuse of expired session tokens
  • Implement sliding timeout mechanisms for better user experience
  • Consider different timeout values for different user roles
  • Log session timeout events for security monitoring

Recommended Timeout Values:

  • High Security: 5-15 minutes of inactivity
  • Standard Security: 15-30 minutes of inactivity
  • Low Security: 30-60 minutes of inactivity

Secure Implementation Example:

// PHP example
session_start();
$timeout_duration = 1800; // 30 minutes

// Check if session has expired
if (isset($_SESSION['last_activity']) && (time() - $_SESSION['last_activity'] > $timeout_duration)) {
    // Session has expired
    session_unset();
    session_destroy();
    header("Location: login.php");
    exit();
}

// Update last activity time
$_SESSION['last_activity'] = time();
